# Task: Audit and Optimize Existing Rizzora Figma Design System

## Goal

Use Figma MCP to inspect the current Rizzora Figma file.

The current file already has initial screens and Foundations.

The task is to audit and optimize the existing design system, variables, styles, and components.

Do not redesign the product screens.

## Step 1: Audit Only

First, inspect the current Figma file and output a report.

Do not modify Figma yet.

Analyze:

1. Current page structure
2. Existing variables
3. Existing color usage
4. Existing text styles
5. Existing effect styles
6. Existing spacing and radius patterns
7. Existing components and variants
8. Auto Layout usage
9. Hardcoded values
10. Frontend handoff risks

## Step 2: Optimization Plan

Create a plan for:

1. Figma page structure
2. Color variable architecture
3. Primitive tokens
4. Semantic tokens
5. Typography styles
6. Spacing variables
7. Radius variables
8. Shadow / glow / blur / gradient styles
9. Component variants
10. Naming conventions
11. Frontend token mapping

## Step 3: Apply After Approval

Only after approval, update the Figma file.

Rules:

- Preserve the existing Rizzora visual direction
- Do not invent a new visual style
- Do not delete existing screens
- Move deprecated elements to 99 Archive
- Use semantic variables in components
- Keep components ready for frontend implementation
